集群部署

我们可以通过单机上的不同端口来模拟多台设备，部署一个伪集群当然实际生产环境不会这样使用。

配置说明

–name
etcd集群中的节点名，不重复就行
–listen-peer-urls
监听的用于节点之间通信的url，可监听多个，集群内部将通过这些url进行数据交互
–initial-advertise-peer-urls
用于节点之间通信的url，节点间将以该值进行通信。
–listen-client-urls
监听客户端通信的url,同样可以监听多个。
–advertise-client-urls
客户端通信url
–initial-cluster-token
etcd集群的初始集群标记。
设置该值后集群将生成唯一id,并为每个节点也生成唯一id,当使用相同配置文件再启动一个集群时，只要该token值不一样，etcd集群就不会相互影响。可以防止运行多个集群时无意中发生的跨集群交互。
–initial-cluster
也就是集群中所有的initial-advertise-peer-urls 的合集
–initial-cluster-state
初始集群状态 (new代表新建)

使用dockerfile构建镜像

发表于 2017-01-11

有时候官方提供的基础镜像不能满足我们的需要，这就要我们自己来构建我们自己的镜像，介绍一下如何使用dockerfile构建自定义镜像。Dockerfile是由一系列命令和参数构成的脚本，这些命令应用于基础镜像并最终创建一个新的镜像。它们简化了从头到尾的流程并极大的简化了部署工作。Docker程序将读取Dockerfile，根据命令生成自定义的镜像。

Dockerfile语法

Dockerfile语法由两部分构成，注释和命令+参数。

Dokcerfile命令

Dockerfile有十几条命令用于构建镜像，下面对这些命令简单的介绍一下。在写Dockerfile命令时不区分大小写，但是建议使用大写。

FROM

FROM命令可能是最重要的Dockerfile命令。命令定义了使用哪个基础镜像启动构建流程。基础镜像可以为任意镜像。如果本地没有该基础镜像，Docker将会从仓库下载该镜像。FROM命令必须是Dockerfile的首个命令。

MAINTAINER

MAINTAINER命令用于声明作者，一般放在FROM的后面。

RUN

构建指令，RUN可以运行基础image支持的命令。如基础镜像选择了ubuntu，那么软件管理部分只能使用ubuntu的命令。

CMD

和RUN命令相似，CMD可以用于执行特定的命令。和RUN不同的是，这些命令不是在镜像构建的过程中执行的，而是在用容器启动时执行的。该指令只能在文件中存在一次，如果有多个，则只执行最后一条。

ENTRYPOINT

指定容器启动时执行的命令，可以多次设置，但是只有最后一个有效。如果你结合CMD命令和ENTRYPOINT命令，你可以从CMD命令中移除“application”而仅仅保留参数，参数将传递给ENTRYPOINT命令。

ADD

ADD命令有两个参数，源和目标。它的基本作用是从源系统的文件系统上复制文件到目标容器的文件系统。如果源是一个URL，那该URL的内容将被下载并复制到容器中。

ENV

在镜像中设置一个环境变量。

VOLUME

VOLUME命令用于让你的容器访问宿主机上的目录。

EXPOSE

EXPOSE用来指定端口，将容器中的端口映射成宿主机器中的某个端口,使容器内的应用可以通过端口和外界交互。

USER

USER命令用于设置运行容器的用户名或UID。默认为root。

WORKDIR

WORKDIR命令用于设置命令的运行目录,对RUN,CMD,ENTRYPOINT生效。

docker常用命令

发表于 2016-12-24

经过了一段时间的使用，对Docker的基础以及常用的命令也了解了一下，现在就总结一下这些常用的命令。

Docker镜像

镜像列表

使用docker images命令可以列出本地主机上的镜像。

查找镜像

使用docker search命令从Docker Hub来查找镜像。

获取镜像

使用docker pull命令来获取一个官方镜像，如果指明tag，比如Ubuntu:15.10，会拉取指定镜像，如果没有指明tag，则默认拉取latest镜像。

我们还可以使用Dockerfile指令来自己创建一个镜像，这个下次再详细介绍。

docker容器

启动容器

启动容器包括基于镜像创建一个容器并启动和启动一个处于stopped状态的已有容器。
新建容器的命令主要使用docker run。

我们可以使用-t -i选项启动一个允许与用户交互的容器，还可以使用-d选项让容器在后台运行，具体各参数用法，大家可以是docker run –help命令查看帮助。
启动一个处于stopped状态的容器命令是docker start命令。

查看正在运行的容器

使用docker ps命令可以列出正在运行的docker容器，使用-a选项还可以列出处于Exited状态即已经退出的容器。

查看容器应用运行日志

使用docker logs 命令来查看应用运行日志。

停止容器

使用docker stop命令停止一个正在运行的容器。

删除应用容器

使用 docker rm命令删除一个容器，该容器必须处于停止状态，否则会报错。

进入容器

如果我们想进入一个在后台运行的容器，有以下几种方法：
1.使用docker attach命令，但是他有一个缺点，当多个窗口同时attach到同一个容器的时候，所有窗口都会同步显示。当某个窗口因命令阻塞时,其他窗口也无法执行操作。而且使用exit命令退出后台后，容器也会退出后台运行。
2.使用docker exec命令。

这样就进入了容器，可以进行操作。

总结

一些常用的docker命令就先介绍这些，docker其实一点都不复杂，而且很轻量，容器都是随用随手创建一个，比用虚拟机好多了。个人感觉还蛮好玩的，有兴趣的可以折腾一下。有时间再总结一下使用dockerfile创建容器，看心情吧。
突然想起今晚是平安夜,祝自己圣诞节快乐！！

docker初探

发表于 2016-12-23

Docker最近貌似很火的样子，本着生命在于折腾的态度，我也来玩一玩。在开始玩了一下之后，感觉Docker还真不错，先看一下Docker到底是什么东西。

Docker是一个开源的应用容器引擎，基于 Go 语言 并遵从Apache2.0协议开源。Docker可以让开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中，然后发布到任何流行的Linux机器上，也可以实现虚拟化。容器是完全使用沙箱机制，相互之间不会有任何接口,更重要的是容器性能开销极低。

下面把docker的配置安装以及常用命令简单介绍一下。本人用的是ubuntu14.04 64位系统，其他版本的linux系统只是安装命令稍有差距。

Docker下载安装

首先Docker要求Ubuntu系统的内核版本高于3.10，你需要先验证你的Ubuntu版本是否支持Docker。ubuntu自带Docker，可以直接安装使用，但是使用的版本较低，所以我们不推荐使用自带的，我们去获取最新的版本（本人可能有强迫症，只要新版不存在重大bug的情况下，都是喜欢用新版本~~）。输入以下命令来获取最新的Docker。wget -qO- https://get.docker.com/ | sh，然后就会下载docker及其依赖包。如果没有错误提示应该就安装成功了。

启动docker后台服务

输入以下命令启动服务，sudo service docker start启动完成后可以使用ps命令查看docker运行情况。然后可以运行hello-world，输入以下命令，docker run hello-world,如果出现如下提示，

Hello from Docker!
This message shows that your installation appears to be working correctly.

说明你的docker已经完美运行起来了，可以愉快进行下一步了。
